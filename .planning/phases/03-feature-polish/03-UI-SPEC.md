## Spacing Scale

All spacing must be multiples of 4. The codebase already uses these values ŌĆö match them exactly.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4pt | Icon gaps, tight inline padding (`padding(.horizontal, 4)`) |
| sm | 8pt | Chip/badge internal spacing, tight element gaps |
| md | 16pt | Default horizontal screen padding, card content padding |
| lg | 24pt | Section padding, modal/sheet horizontal insets |
| xl | 32pt | Empty state text horizontal padding |
| 2xl | 48pt | Bottom padding on auth screens, empty state icon size offset |
| 3xl | 64pt | Not used in Phase 3 features |

Exceptions:
- Swipe action buttons: HStack spacing 44pt (existing established value ŌĆö do not change)
- Card height: 580pt fixed (existing established value ŌĆö do not change)
- Card corner radius: 24pt (existing established value ŌĆö do not change)
- Touch targets for action buttons (accept/reject circles): 76pt outer / 64pt inner (existing established value ŌĆö do not change)
- Minimum tappable area for all other interactive elements: 44pt (iOS HIG)

## Interaction Contracts

### Swipe Card Animation

- **Fly-off:** `withAnimation(.easeOut(duration: 0.35))` ŌĆö drag offset ┬▒600pt on x-axis. Existing pattern in `JobCardView`. Do not change animation curve or duration.
- **Snap-back:** `withAnimation(.spring(response: 0.4, dampingFraction: 0.7))` ŌĆö drag offset reset to `.zero`. Do not change.
- **Button-triggered fly-off (accept/reject buttons):** Use identical `.easeOut(duration: 0.35)` animation with `Task.sleep(nanoseconds: 350_000_000)` before calling `onSwipe`. Existing pattern ŌĆö preserve it.
- **manualInputFields bug fix:** When `manualInputFields` is non-empty on accept, do NOT reset `dragOffset = .zero` before the sheet appears. Defer the reset until the sheet is dismissed and the answer callback fires.
- **Haptic:** `UIImpactFeedbackGenerator(style: .medium)` on accept, `.light` on reject. Fires in `swipeJob()` in `ApplyView` ŌĆö existing, verify it fires once per swipe, not twice.

### Swipe Limit Banner

- **Threshold:** 5 swipes remaining (existing: toolbar badge changes color at `<= 5`)
- **Appearance:** Existing toolbar capsule badge (`bolt.fill` icon + "X left" text). No additional persistent banner needed ŌĆö the toolbar badge is sufficient. Polish: ensure it is always visible when `swipesRemaining != nil`, not just at threshold.
- **At limit (noSwipesLeft = true):** Show `noSwipesView` full-screen replacing the card deck. Existing structure present ŌĆö verify upgrade CTA links to `PremiumView`.

### isGreatFit / greatMatch Badge

- **Position:** Top-right of card header, using existing `matchBadge()` helper.
- **Visual:** Capsule with `#00c97a` background, white text + `star.fill` SF Symbol icon, 9pt bold text, 8pt horizontal padding, 5pt vertical padding. Existing `matchBadge()` renders this ŌĆö do not redesign.
- **Logic:** Check BOTH `job.greatMatch == true` AND `job.isGreatFit == true` (two separate model fields). Render one badge if either is true. Label: "Great Fit" (not "Great Match" ŌĆö update string if only `isGreatFit` is set, keep "Great Match" if only `greatMatch` is set).

### Pipeline Stage Moves

- **Trigger:** Tap job row in `MyJobsView` ŌåÆ bottom sheet (`JobDetailSheet`).
- **Stage selector:** Row of labeled buttons for each of the 7 stages inside the sheet. Active stage highlighted with its `PipelineStage.color`. Inactive stages use `.secondary` text on white background.
- **Optimistic update:** Move the job locally in the ViewModel array before the API call resolves. Revert silently if the call fails ŌĆö show an inline error banner (not an alert).
- **Active-only default view:** Show 5 active stages by default: Applying, Applied, Screening, Interview, Offer. Toggle to show all 7 (adds Archived and Failed). Toggle is a segmented control or labeled button at the top of `MyJobsView`. Claude's discretion: use a segmented control labeled "Active" / "All".

### Payment Flow

- **Stripe return detection:** Observe `scenePhase == .active` in `MainTabView` or `AppRouter`. When scene becomes active AND `SubscriptionViewModel.pendingSessionId != nil`, call `checkSessionStatus(sessionId:)` automatically.
- **`pendingSessionId` storage:** Store the `session_id` query parameter extracted from `checkoutURL` in `SubscriptionViewModel` before opening `SFSafariViewController`.
- **"Verifying payment..." overlay:** Full-screen overlay on `PremiumView` using `LoadingView(message: "Verifying payment...")`. Show when `isVerifyingPayment == true`. Dismiss when `checkSessionStatus` completes.
- **Post-verification:** Show success message "Plan updated! You're now on [plan name]." as a non-modal banner that auto-dismisses after 3 seconds. On failure: inline error banner with "Verification failed ŌĆö please contact support."

### Loading States

- **Main content areas** (full-screen fetch in progress): Replace generic `ProgressView()` with branded logo animation. Implementation: `Image("FlashApplyLogo")` with a repeating scale pulse animation ŌĆö `withAnimation(.easeInOut(duration: 0.9).repeatForever(autoreverses: true)) { scale = 1.1 }`. Use `LoadingView` as the wrapper, replacing its current `ProgressView()` body.
- **Quick inline actions** (save profile, move pipeline stage): Inline `ProgressView()` at `.scaleEffect(1.0)` tinted `.flashTeal`. No full-screen takeover.
- **Apply screen:** Existing `LoadingView(message: "Finding jobs for you...")` ŌĆö replace spinner with branded animation.

### Error Display

- **Severity-based approach** (Claude's discretion decision):
  - **Transient / recoverable errors** (network timeout, save failure): Inline banner at the top of the screen. Background: `Color(hex: "#e74c3c").opacity(0.10)`. Border: `Color(hex: "#e74c3c").opacity(0.3)`. Text: `.flashDark` at 14pt. Includes a "Retry" button. Auto-dismisses after 5 seconds or on retry tap.
  - **Blocking errors** (auth failure, fatal data load failure): `ErrorView` full-screen replacement with retry button. Existing component ŌĆö ensure it always has a retry action passed in.
  - **Destructive confirmation**: Use `.confirmationDialog` (SwiftUI native action sheet). Never `Alert` for destructive actions.

- **Error message mapping** (for UX-01): Map raw SDK strings before assigning to `@Published var error: String?`. Mapping table:

| Raw pattern | Human-readable message |
|-------------|------------------------|
| `"UserNotConfirmedException"` | "Please verify your email before signing in." |
| `"NotAuthorizedException"` | "Incorrect email or password. Please try again." |
| `"UsernameExistsException"` | "An account with this email already exists." |
| `"NetworkError"` / `"URLError"` | "No internet connection. Check your connection and retry." |
| `"403"` / `"Forbidden"` | "You've reached your daily limit. Upgrade to continue." |
| `"404"` / `"Not Found"` | "This content is no longer available." |
| `"500"` / `"Internal Server Error"` | "Something went wrong on our end. Please try again." |
| Anything else | "An unexpected error occurred. Please try again." |

## Phase-Specific Constraints

- **iOS 16.0+ only.** All SwiftUI APIs must be available on iOS 16. No iOS 17+ APIs (e.g., `@Observable`, `.scrollPosition`).
- **No StoreKit / Apple IAP** in this phase. Stripe web checkout via `SFSafariViewController` is the only payment path.
- **No `appliedDate` on pipeline cards.** Backend model has no field. Do not show placeholder text ŌĆö omit entirely.
- **No drag-and-drop pipeline stage moves.** Stage moves via detail sheet buttons only.
- **No disk image caching.** `AsyncImage` / `CompanyLogoView` remains as-is. Image caching is Phase 4.
- **`seenUrls` unbounded growth.** Do not cap in this phase. Phase 4 concern.
